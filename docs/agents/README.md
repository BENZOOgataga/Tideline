# Agent docs

Documentation aimed at any AI agent (or human) picking up this codebase. Read in this order:

1. [`HANDOFF.md`](HANDOFF.md) -> what state the project is in and how to build, run, and test.
2. [`ARCHITECTURE.md`](ARCHITECTURE.md) -> module boundaries and the key cross-module flows.
3. [`COMMON_ISSUES.md`](COMMON_ISSUES.md) -> traps already paid for. Check here when something feels weird.
4. [`RELEASING.md`](RELEASING.md) -> how releases work. Agents do not initiate them.

The canonical product design lives in [`../plans/SPEC.md`](../plans/SPEC.md). The agent rules of engagement live in [`../../CLAUDE.md`](../../CLAUDE.md).
